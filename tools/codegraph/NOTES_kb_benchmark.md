# CodeGraph fork — kb_benchmark vendoring notes

This directory is a **vendored copy** of the CodeGraph fork with
Verilog/SystemVerilog support added. It is NOT a git submodule and
does NOT contain its own `.git/` — the upstream + fork history was
preserved as patch files under `_patches/`.

## Provenance

- Upstream: `https://github.com/colbymchenry/codegraph`
- Upstream base commit: `4077ed19b7d8a88eba93601c0c308e59c8640f8c` (v1.1.1)
- Fork branch (in original local-only worktree): `feat/verilog-language-module`
- Vendored into kb_benchmark on: 2026-06-30

## How to (re)build for use

```bash
cd tools/codegraph
npm install               # ~1-2 minutes; installs dev + runtime deps
npm run build             # produces dist/bin/codegraph.js
```

The `node_modules/` and `dist/` directories are gitignored (per the
upstream `tools/codegraph/.gitignore` we vendored) — install + build
is required on every fresh checkout.

## Runtime requirement

CodeGraph requires **Node ≥ 22.5** for `node:sqlite`. On macOS with
Homebrew, use `/opt/homebrew/opt/node@22/bin/node` explicitly when
invoking `dist/bin/codegraph.js`.

## How to upgrade upstream later

When picking up a new CodeGraph upstream release:

```bash
# Save a safety tag in kb_benchmark
git -C /path/to/kb_benchmark tag pre-codegraph-upgrade-$(date +%Y%m%d) HEAD

# Re-clone upstream at the new tag into a scratch dir
git clone https://github.com/colbymchenry/codegraph /tmp/cg-new
cd /tmp/cg-new
git checkout <new-upstream-sha>

# Re-apply the three Verilog commits
git am /path/to/kb_benchmark/tools/codegraph/_patches/000*.patch
# Resolve conflicts, commit, record the new base sha

# rsync into kb_benchmark (preserve our gitignore exclusions)
rsync -a --exclude=.git --exclude=node_modules --exclude=dist \
  --exclude=docs --exclude=site --exclude=assets \
  --exclude=telemetry-worker \
  /tmp/cg-new/ /path/to/kb_benchmark/tools/codegraph/

# Rebuild + re-test inside kb_benchmark
cd /path/to/kb_benchmark/tools/codegraph
npm install && npm run build
npx vitest run __tests__/verilog-extractor.test.ts
```

After upgrading, also regenerate the three `_patches/` files to reflect
any conflict resolutions, and update the "upstream base commit" sha at
the top of this NOTES file.
